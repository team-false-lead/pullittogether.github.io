Project Overview

Pull It Together is an online cooperative survival adventure where teams haul a wagon through a procedurally generated wilderness with dynamic encounters while outrunning a massive superstorm.

This page serves as an overview of our major systems with highlights on the team stress director, procedural generation pipeline, multiplayer replication framework, and more! For more information on the design and development process:

  • Meet The Team
  • Core concept: cooperative traversal + survival under escalating chase pressure
  • Our focus: adaptive pacing, deterministic procedural worlds, multiplayer stability, cooperative coordination
  • Tech: Godot + C#/GDScript/Shaders + custom tools/pipelines

Team Stress Director &
Dynamic Encounter Systems

Adaptive pacing director that evaluates team condition vs a target difficulty curve and selects encounters to maintain an escalating, fair chase.

  • Stress inputs: health/stamina/down states, wagon condition, inventory/mitigation, storm proximity, recent encounters, terrain severity
  • Selector: weighted encounter pool (attacks, recovery moments, weather events, “wacky” encounters)
  • Storm adaptation: speed and escalation remains reactive without hard spikes

Procedural Trail &
Terrain Generation Pipeline

Deterministic generation pipeline that produces unique runs from a single seed, with trail difficulty tied to the predefined target curve.

  • Trail: difficulty-profiled segments, constrained noise (elevation/steepness/curvature), smoothing for playability
  • Terrain: shader-based biome blending, constraints from trails/rivers/key areas, deterministic scatters
  • Multiplayer: host-baked heightmap shared and reconstructed across clients

Multiplayer Item Replication &
Authority Framework

Host-authoritative framework for items, entities, and dynamic events: spawning, ownership, movement/state replication, and stable interactions.

  • Auto registration: ItemSpawnRegistry scans folders and prepares/registers .tscn items for designers
  • Deterministic IDs: synchronized dictionaries prevent desync and ghost references
  • Validated interactions: pickup/drop/inventory/rope/tool usage run through host logic
  • Proxies: local visual proxies reduce jitter for held items
  • Join-in-progress: full state reconstruction for new peers
  • Cleanup: authority resets, proxies freed, dictionaries trimmed on disconnect

Networked Synchronized
Animation System

Multiplayer animation layer that replicates parameter values (not frames) so clients reconstruct motion locally with identical animation graphs.

  • Replicated values: movement blend vectors, interaction indices, action speed multipliers
  • State blends: aiming, inventory state, item usage blends
  • Benefits: low bandwidth, predictable interactions, stable group animations

Additional Systems

  • Gameplay: inventories/slotting, rope heaving/tethering, harvesting, interaction frameworks
  • NPC AI: animal behavior trees and encounter behaviors
  • Tech art: terrain heightmaps, storm clouds, weather event visuals, biome blending
  • Multiplayer/backend: Steamworks integration, menu flow, plugin integration, custom engine build
  • Team support: debugging/refactoring, tools for iteration, performance profiling, stability improvements
